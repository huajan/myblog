先把重点告诉你，如果你也发现jekyll执行的时候报错是Tag was never closed
那么在你的blog的_config.yml文件中，加上这句：excerpt_separator: ""
就可以了。

没想到ubuntu上装起来也这么多坑！ubuntu版本12.10；
在装jekyll之前要装，ruby，rubygem，rdoc，ruby-dev;
{%highlight java%}
sudo apt-get install ruby
{%endhighlight%}
然后这里下载个rubygem，解压后cd进目录，执行
{%highlight ruby%} ruby setup.rb
{%endhighlight%}
ok继续执行
{%highlight ruby%} sudo apt-get install ruby1.9.1-dev sudo gem install rdoc sudo gem install jekyll //安装jekyll
{%endhighlight%}

好吧，装好这些，先搞个测试，执行 jekyll new blog ，进到blog目录，启动jekyll，在blog的根目录执行命令：jekyll serve
没报错的话，浏览器查看：localhost:4000，如果出来jekyll的demo就ok了。

如果报错，上网查查看是啥错吧，
